# 플래닉(Planic) 정보구조(IA) 및 권한 구조

사용자 화면과 관리자 화면을 명확히 분리한 경로·역할 정의 문서입니다.

---

## 1. 분류 기준

- **사용자 화면**: 고객이 서비스를 사용해 결과를 얻는 흐름 (로그인, 생성, 내 문서, 결제, 설정, 이력)
- **관리자 화면**: 운영자가 서비스를 통제·조정하는 흐름 (사용자/구독/플랜/사용량/엔진/로그/시스템 관리)

---

## 2. 사용자 영역 (일반 경로)

| 경로 | 분류 | 설명 |
|------|------|------|
| `/` | 사용자 | 랜딩·소개 |
| `/auth` | 사용자 | 로그인 / 회원가입 (Google) |
| `/plans` | 사용자 | 플랜 선택 (고객용 요금제 보기) |
| `/generate` | 사용자 | 견적·기획안 생성 (메인 작업 화면) |
| `/prices` | 사용자 | 단가표 관리 (내 단가 → AI 반영) |
| `/references` | 사용자 | 참고 견적서·시나리오·과업지시서·큐시트 샘플 |
| `/settings` | 사용자 | 회사 정보·기본 견적 설정 |
| `/history` | 사용자 | 견적 이력·통계 |

**사용자 네비게이션(GNB)**: 견적, 참고, 단가, 이력, 설정 — 관리자 링크 없음.

---

## 3. 관리자 영역 (/admin 이하)

| 경로 | 분류 | 설명 |
|------|------|------|
| `/admin` | 관리자 | 관리자 로그인 또는 대시보드 |
| `/admin/users` | 관리자 | 사용자 관리 (준비 중) |
| `/admin/subscriptions` | 관리자 | 구독 관리 (준비 중) |
| `/admin/plans` | 관리자 | 플랜 관리 (준비 중) |
| `/admin/usage` | 관리자 | 사용량 관리 (준비 중) |
| `/admin/engines` | 관리자 | 엔진/모델 관리 (준비 중) |
| `/admin/logs` | 관리자 | 로그 (준비 중) |
| `/admin/system` | 관리자 | 시스템·공지 (준비 중) |

**권한**: 관리자 쿠키(`planic_admin`)가 있는 경우에만 `/admin` 및 `/admin/*` 접근 가능.  
`/admin/xxx`(하위 경로)는 쿠키 없이 접근 시 `/admin`(로그인)으로 리다이렉트.

**관리자 전용 UI**: `/admin` layout에서 로그인 시에만 사이드바(AdminShell) 노출.  
사이드바 메뉴: 대시보드, 사용자 관리, 구독 관리, 플랜 관리, 사용량 관리, 엔진/모델, 로그, 시스템 + 메인으로 / 로그아웃.

---

## 4. 관리자 메뉴 후보 (향후 구현)

- **사용자 관리**: 가입 사용자 목록, 권한, 정지
- **구독 관리**: 구독 현황, 갱신, 해지, 결제 이력
- **플랜 관리**: 요금제·한도·기능 설정
- **사용량 관리**: API·생성 사용량, 한도 초과 처리
- **엔진/모델**: AI 모델 선택·엔진 정책
- **프롬프트/정책**: 시스템 프롬프트·정책 관리 (필요 시)
- **로그**: 요청·장애·감사 로그
- **시스템**: 공지, 점검, 시스템 설정

---

## 5. 애매했던/추후 정리 영역

- **`/plans`**: 고객용 "플랜 보기"이므로 사용자 영역으로 유지.  
  요금제·한도 **운영**은 `/admin/plans`에서 할 예정.
- **참고 견적서·단가표·이력**: 모두 "내 데이터"이므로 사용자 영역.  
  관리자 대시보드에서는 이전처럼 사용자 페이지로의 바로가기를 두지 않음 (역할 분리).

---

## 6. 참고

- 관리자 인증: `lib/admin-auth.ts`, 쿠키 기반 세션
- 미들웨어: `middleware.ts` — `/admin/:path*` 에서 쿠키 없으면 `/admin` 리다이렉트
- 레이아웃: `app/admin/layout.tsx` — 세션 있을 때만 `AdminShell`(사이드바) 적용
