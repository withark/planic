# Google OAuth — 플래닉 브랜딩 (쿼닥 등 구 앱 이름 제거)

구글 로그인 중간 화면에 **앱 이름·로고**가 뜨는 위치는 **앱 코드가 아니라** Google Cloud Console의 **OAuth 동의 화면**입니다.  
예전에 **쿼닥** 등으로 만든 클라이언트/동의 화면을 그대로 쓰면, 사용자에게 계속 그 이름이 보입니다.

## 1. 지금 쓰는 `GOOGLE_CLIENT_ID`가 어느 프로젝트 소속인지 확인

1. **Vercel(또는 배포 환경)** → 프로젝트 → Settings → Environment Variables 에서 `GOOGLE_CLIENT_ID` 값을 복사합니다.  
   로컬이면 `.env.local` 의 값.
2. 클라이언트 ID 형식:  
   `{숫자}-{랜덤문자}.apps.googleusercontent.com`  
   앞의 **숫자 구간**은 보통 해당 GCP 프로젝트와 연결된 식별 정보입니다.
3. [Google Cloud Console](https://console.cloud.google.com/) 접속 → 상단 프로젝트 선택기로 **예전 쿼닥·공용**으로 쓰던 프로젝트를 엽니다.
4. **API 및 서비스** → **사용자 인증 정보** → **OAuth 2.0 클라이언트 ID** 목록에서, 위에서 복사한 ID와 **동일한 클라이언트**가 있는 프로젝트가 곧 **현재 자격 증명이 속한 프로젝트**입니다.

> 한 프로젝트 안에 클라이언트가 여러 개면, ID 문자열 전체가 일치하는 행을 찾으면 됩니다.

## 2. 플래닉 이름으로만 바꾸는 경우 (같은 프로젝트 유지)

쿼닥 전용 서비스가 더 이상 없고, **이 GCP 프로젝트를 플래닉만** 쓸 계획이면:

1. 같은 프로젝트에서 **API 및 서비스** → **OAuth 동의 화면**.
2. **앱 정보** 편집:
   - **앱 이름**: 예) `플래닉` 또는 `Planic` (사용자에게 보이는 문구).
   - **사용자 지원 이메일** / **앱 로고**(선택): 플래닉 기준으로 통일.
3. 저장 후, 변경이 반영되기까지 **수 분~최대 하루** 걸릴 수 있습니다.  
4. **클라이언트 ID/Secret 은 그대로** 두고, Vercel의 `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` 는 수정할 필요 없습니다.

## 3. 쿼닥과 분리 — 플래닉 전용 OAuth 클라이언트로 재발급 (권장 구조)

쿼닥과 **같은 프로젝트/같은 클라이언트**를 쓰지 않으려면:

### A. 새 GCP 프로젝트를 플래닉 전용으로 생성 (권장)

1. Cloud Console → **새 프로젝트** → 이름 예: `Planic Production`.
2. **OAuth 동의 화면**:
   - 사용자 유형: 외부(테스트 시 테스트 사용자 추가) 또는 내부.
   - **앱 이름**: `플래닉` / `Planic`.
   - 개발자 연락처 이메일 등 플래닉 기준으로 입력.
3. **사용자 인증 정보** → **사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**:
   - 애플리케이션 유형: **웹 애플리케이션**.
   - 이름: 예) `Planic Web`.
   - **승인된 리디렉션 URI** (플래닉 NextAuth 기준):
     - `https://www.planic.cloud/api/auth/callback/google`
     - `https://planic.cloud/api/auth/callback/google` (apex 대응)
     - 로컬: `http://localhost:3000/api/auth/callback/google`
4. 생성 후 **클라이언트 ID** / **클라이언트 보안 비밀번호** 복사.

### B. 배포 환경 변수 교체

| 변수 | 조치 |
|------|------|
| `GOOGLE_CLIENT_ID` | 새 웹 클라이언트 ID로 교체 |
| `GOOGLE_CLIENT_SECRET` | 새 비밀번호로 교체 |

- **Vercel** Production(및 Preview 필요 시) 환경 변수 갱신 후 **재배포**.
- 로컬 `.env.local` 도 동일 값으로 맞춤.

### C. 쿼닥 쪽

- 쿼닥 서비스가 **별도 서버**에서 예전 Client ID 를 쓰고 있다면, 쿼닥은 **기존 프로젝트·기존 클라이언트**만 두고 플래닉은 **새 프로젝트**로 두면 완전히 분리됩니다.

## 4. 앱 검증(Verification)

- 동의 화면에 **민감/제한 범위**만 쓰고 **일반 프로필(email 등)** 정도면, 테스트 사용자로 충분한 경우가 많습니다.
- **프로덕션**으로 전면 공개하려면 Google의 앱 검증 절차가 필요할 수 있습니다. [OAuth 검증](https://support.google.com/cloud/answer/9110914) 문서를 참고하세요.

## 5. 이 저장소에서의 역할

| 항목 | 설명 |
|------|------|
| 앱 이름 **쿼닥** 표시 | 코드 수정으로는 바뀌지 않음. **동의 화면 앱 이름** 또는 **새 프로젝트 + 새 동의 화면**으로 해결. |
| `GOOGLE_CLIENT_ID` | 어느 클라이언트를 가리키는지만 바꾸면 됨 (플래닉 전용 발급 시 새 값). |

자세한 환경 변수 목록은 [AUTH_ENV.md](./AUTH_ENV.md) 를 참고하세요.
