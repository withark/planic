# 관리자 기능 재정의 및 UI 설계

## 1. 현재 샘플 관리 문제점

- **역할 불명확**: "샘플 관리"라는 이름만으로는 파일 업로드 메뉴인지, 생성 품질 설정인지 구분이 안 됨.
- **업로드 후 결과 부재**: 업로드 후 어떤 문서 탭에 연결되는지, 생성 결과에 어떻게 반영되는지 설명이 없음.
- **파싱 결과 미리보기 없음**: 추출된 구조(제목/섹션/표/항목)를 운영자가 확인할 수 없음.
- **반영 방식 미설명**: 문체 참고 / 구조 참고 / 레이아웃 참고 / 표 형식 참고 등이 UI에 없고, "예상 영향" 문구도 없음.
- **문서 유형 제한**: 큐시트·제안·타임테이블·시나리오만 있고, 견적서·과업지시서·제안서 등이 없음.
- **흐름 부재**: "업로드 → 분석 → 연결 → 반영 방식 확인"이 한 화면에서 이어지지 않음.

---

## 2. 현재 엔진 강화 설정 문제점

- **이름이 추상적**: "엔진 강화"만으로는 무엇을 강화하는지, 운영자가 무엇을 바꿔야 하는지 이해 불가.
- **역할 혼재**: provider/model/maxTokens(인프라)와 구조/문체/품질(생성 규칙)이 한 덩어리로만 노출됨.
- **탭별 규칙 없음**: 견적서/제안 프로그램/타임테이블/큐시트/시나리오별로 다른 규칙을 줄 수 없음.
- **샘플 반영 강도 불명확**: "샘플 가중치·반영 지시" 텍스트 필드만 있어서 약/보통/강 선택이 없음.
- **우선 기준 선택 없음**: 문체 우선/구조 우선/레이아웃 우선 등이 체크박스 2개(구조·문체)로만 표현됨.
- **출력 형식 선택 없음**: 문단형/표형/혼합형/운영문서형 등이 없고 자유 텍스트만 있음.
- **실제 반영 확인 불가**: Claude API 요청 시 이 설정이 어떻게 주입되는지 화면에서 확인할 수 있는 구조가 없음.

## 3. 더 이해하기 쉬운 메뉴명 제안

| 기존 메뉴명 | 제안 메뉴명 | 비고 |
|------------|------------|------|
| 샘플 관리 | **기준 양식 관리** | 문서 생성 시 참고할 기준 양식을 등록·연결하는 기능임을 드러냄 |
| 엔진 강화 설정 | **생성 규칙 설정** | 문서를 어떤 기준으로 생성할지 조정하는 기능임을 드러냄 |
| (신규) | **외부 자료 수집** | 웹/URL 기반 자료 수집 → 검토 → 샘플 등록 흐름 |

대안: "문서 생성 강화 설정", "샘플 반영 규칙" (생성 규칙 설정의 하위 개념으로 사용 가능)

## 4. 샘플 업로드 후 보여줄 정보 구조

업로드(및 수정) 후 운영자에게 **반드시** 노출할 정보:

| 항목 | 설명 | 비고 |
|------|------|------|
| 파일명 | 원본 파일명 | |
| 문서 유형 | 견적서 / 제안 프로그램 / 타임테이블 / 큐시트 / 시나리오 / 과업지시서 / 제안서 | 선택 |
| 추출된 구조 | 제목, 섹션, 표, 항목 등 파싱 결과 요약 | 미리보기/펼침 |
| 반영 대상 탭 | 견적서 내 어떤 탭(큐시트·시나리오 등)에 쓰일지 | |
| 반영 방식 | 문체 참고 / 구조 참고 / 레이아웃 참고 / 표 형식 참고 (복수 선택 가능) | |
| 예상 영향 설명 | 예: "향후 큐시트 생성 시 시간/담당/준비물 열 구성을 우선 반영합니다" | 운영자 입력 또는 자동 문구 |
| 우선순위 | 숫자 (높을수록 우선 반영) | 기존 유지 |
| 활성/비활성 | 생성 시 사용 여부 | 기존 유지 |
| 설명 | 메모 | 기존 유지 |
| 최근 사용 횟수 | generation_use_count | 기존 유지 |
| 최근 반영 이력 | last_used_at 또는 생성 로그 링크 | 기존 + 로그 연동 |

**흐름**: 업로드 → (서버) 분석·파싱 → 연결 탭·반영 방식 설정 → 예상 영향 확인 → 저장.

## 5. 생성 규칙 설정의 실제 항목 구조

운영자가 "어떤 규칙으로 문서가 생성되는지" 직관적으로 이해할 수 있도록 구성.

### 5.1 인프라 (기존 유지, 상단 요약)

- provider / model / maxTokens (env + DB 오버레이)

### 5.2 탭별 생성 규칙 (탭 = 견적서 내 탭 또는 문서 유형)

| 탭 | 설정 가능 항목 예시 |
|----|---------------------|
| 견적서 | 출력 형식, 표 우선 여부 |
| 제안 프로그램 | 문체/구조 우선 |
| 타임테이블 | 시간 단위, 표 형식 |
| 큐시트 | 열 구성(시간/담당/준비물 등) 우선 |
| 시나리오 | 문단형/연출 메모 형식 |

(초기에는 "공통 규칙" 하나로 두고, 추후 탭별로 분리 가능)

### 5.3 샘플 반영 강도

- **약하게** / **보통** / **강하게** (라디오 또는 셀렉트)
- 저장 시 `sampleWeightNote` 또는 별도 필드로 프롬프트에 주입

### 5.4 우선 기준 (단일 또는 복수)

- 문체 우선
- 구조 우선
- 레이아웃 우선
- 표 형식 우선
- 항목 구성 우선

(기존 structureFirst / toneFirst 확장)

### 5.5 출력 형식

- 문단형
- 표형
- 혼합형
- 운영문서형

(기존 outputFormatTemplate을 선택 + 보조 텍스트로 확장 가능)

### 5.6 기본 생성 포맷

- JSON 구조 힌트, 필수 필드 등 (고급용, 선택)

### 5.7 Claude API 요청 시 실제 반영 확인

- "이 설정이 프롬프트에 어떻게 들어가는지" 미리보기 블록 (buildGeneratePrompt 요약 또는 샘플 문자열 노출)

## 6. 외부 자료 수집 기능 구조

**원칙**: 외부 자료는 **바로 생성에 쓰지 않고**, 운영자가 검토 후 **샘플로 등록**하는 구조.

### 6.1 메뉴명

- **외부 자료 수집** (권장) / 웹 샘플 수집 / 레퍼런스 수집

### 6.2 필수 기능

| 기능 | 설명 |
|------|------|
| 키워드 기반 자료 수집 | 검색어로 웹/문서 후보 수집 (검색 API 또는 크롤링 정책에 따름) |
| URL 등록 기반 수집 | URL 입력 시 해당 페이지/문서 수집 후보로 추가 |
| 문서 후보 목록 보기 | 수집된 후보 목록 (제목, URL, 유형, 수집일) |
| 문서 미리보기 | 텍스트/구조 미리보기 |
| 문서 유형 지정 | 견적서/제안서/과업지시서/행사 기획서/운영계획서/큐시트/타임테이블/시나리오 등 |
| 샘플로 저장 | 후보를 "기준 양식"으로 저장 → 기준 양식 관리 테이블로 넘김 |
| 기준 양식 관리로 넘기기 | 저장 시 기준 양식 관리에 동일 메타(유형, 탭, 반영 방식 등)로 등록 |

### 6.3 수집 대상 문서 유형 예시

- 견적서, 제안서, 과업지시서, 행사 기획서, 운영계획서, 큐시트, 타임테이블, 시나리오

### 6.4 흐름

1. **수집**: 키워드 검색 또는 URL 등록 → 후보 목록 생성
2. **검토**: 목록에서 미리보기 → 유형 지정, 필요 시 편집
3. **등록**: "샘플로 저장" 클릭 → 기준 양식 관리에 등록 (연결 탭, 반영 방식 등 설정)
4. 생성 파이프라인은 **기준 양식 관리**에만 의존하므로, "외부 자료 수집"은 입력 소스일 뿐.

## 8. 필요한 DB 필드 / 상태값 제안

### 8.1 cuesheet_samples (기준 양식) 확장 제안

| 필드 | 타입 | 비고 |
|------|------|------|
| (기존) id, user_id, filename, ext, uploaded_at, display_name, document_tab, description, priority, is_active, archived_at, generation_use_count, last_used_at | | 유지 |
| reflection_methods | text[] 또는 jsonb | 문체/구조/레이아웃/표 형식 참고 (복수) |
| expected_impact | text | 예상 영향 설명 문구 |
| parsed_structure_summary | text | 파싱 결과 요약 (제목/섹션/표 등) |
| document_type | text | 견적서/제안서/과업지시서/… (document_tab과 매핑 또는 확장) |

### 8.2 engine_config (KV) 확장 제안

| 키 | 비고 |
|----|------|
| sampleReflectionStrength | 'low' \| 'medium' \| 'high' |
| priorityCriteria | ['tone' \| 'structure' \| 'layout' \| 'table' \| 'items'] |
| outputFormat | 'paragraph' \| 'table' \| 'mixed' \| 'operational' |
| tabRules | 탭별 오버레이 (선택, 추후) |

### 8.3 외부 자료 수집 (신규 테이블 제안)

| 테이블 | 용도 |
|--------|------|
| reference_candidates | 수집된 URL/키워드 결과, 제목, 수집일, 상태(pending/reviewed/registered/discarded) |
| reference_candidate_content | 미리보기용 텍스트/메타 (또는 파일 스토리지) |

상태값: `pending` → 검토 대기, `reviewed` → 검토 완료, `registered` → 기준 양식으로 등록됨, `discarded` → 제외.
